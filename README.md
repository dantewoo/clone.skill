# 克隆.skill 架构说明
<img width="280" height="220" alt="image" src="https://github.com/user-attachments/assets/06c181a6-b31e-4e24-837d-5c843d61f408" />

> 投喂资料，一键将名人、前任、现任、仇敌、兄弟、姐妹等等影响你的人，变成 AI 数字人


## 这是什么

你只需要做一件事——**丢资料进去**。

聊天记录、书籍、日记、文章、社交媒体……clone.skill 会自动分析这个人的语言习惯、思维方式、情绪模式和生活状态，最终生成一个可以对话的 AI 克隆体。

不是角色扮演，不是关键词模仿。是 **1:1 的人格蒸馏**。

## 支持的文本类型

### 1. 聊天记录
- **格式**：微信/QQ/Telegram 导出的 txt
- **解析器**：`chat_parser.py`
- **特点**：最适合全面分析，数据最充分
- **优势**：能提取语言习惯、表达节奏、真实社交场景

### 2. 书籍
- **格式**：txt/md/pdf
- **解析器**：`text_parser.py`
- **特点**：适合分析思想深度、专业能力、价值观
- **注意**：作者的"写作人格"可能与"日常人格"不同

### 3. 文章/博客
- **格式**：txt/md
- **解析器**：`text_parser.py`
- **特点**：适合分析专业领域、表达风格
- **优势**：能看到公开表达的观点和立场

### 4. 日记
- **格式**：txt/md
- **解析器**：`text_parser.py`
- **特点**：最真实的内心世界
- **优势**：能看到情绪变化、真实想法、人生目标

### 5. 社交媒体内容
- **格式**：txt（导出的微博/推特/朋友圈等）
- **解析器**：`text_parser.py`
- **特点**：适合分析公开人格、社交风格
- **注意**：社交媒体上的表达可能经过修饰

### 6. 邮件
- **格式**：txt/eml
- **解析器**：`text_parser.py`
- **特点**：适合分析工作沟通、正式表达
- **优势**：能看到职业化的沟通方式

### 7. 其他任意文本
- **格式**：txt/md
- **解析器**：`text_parser.py`
- **特点**：根据内容灵活提取特征
- **注意**：数据不足的维度标注"无数据"

## 工具链

### chat_parser.py
- **用途**：解析聊天记录
- **输入**：微信/QQ/Telegram 导出的 txt
- **输出**：带时间权重的 JSON + Markdown
- **特点**：自动计算时间权重、统计回复速度

### text_parser.py（新增）
- **用途**：解析通用文本
- **输入**：任意 txt/md 文件
- **输出**：统一格式的 JSON + Markdown
- **特点**：自动识别文本类型（chat/book/article/diary/other）

## 使用场景

### 场景 1：克隆真实朋友/同事
- **推荐文本**：聊天记录
- **数据量**：建议 500+ 条消息
- **效果**：语言习惯、性格特征最准确

### 场景 2：克隆历史人物/作家
- **推荐文本**：书籍、文章、日记
- **数据量**：建议 5000+ 字
- **效果**：思想深度、价值观最准确
- **注意**：生活块数据可能不足

### 场景 3：克隆公众人物
- **推荐文本**：社交媒体内容、采访、文章
- **数据量**：建议 3000+ 字
- **效果**：公开人格、表达风格准确
- **注意**：私下人格可能不同

### 场景 4：克隆虚拟角色
- **推荐文本**：角色对话、设定文档
- **数据量**：建议 2000+ 字
- **效果**：语言风格、性格设定准确
- **注意**：需要明确标注为虚拟角色

---

## 数据质量要求

### 最低要求
- 聊天记录：100+ 条消息
- 书籍/文章：2000+ 字
- 日记：1000+ 字，跨度 30+ 天
- 其他文本：1000+ 字

### 推荐标准
- 聊天记录：500+ 条消息，跨度 90+ 天
- 书籍/文章：5000+ 字
- 日记：3000+ 字，跨度 90+ 天
- 其他文本：3000+ 字

### 数据不足时
- 自动标注置信度（高/中/低）
- 数据不足的维度标注"无数据"
- 提示用户追加更多资料
