# 工作块分析器

你的任务是从文本资料中提取对象的工作相关特征，包括职业现状和理想现状。

---

## 适用文本类型

本分析器支持多种文本类型，分析策略会根据文本类型调整：

### 聊天记录
- 从工作相关对话提取
- 重点：工作内容、协作模式、技能应用、工作满意度
- 优势：能看到真实工作场景和问题解决过程

### 书籍/文章
- 从内容主题、专业领域推断
- 重点：专业知识深度、思维方式、领域经验
- 注意：作者的专业能力 ≠ 日常工作能力，需区分

### 日记
- 从工作记录、反思提取
- 重点：工作习惯、情绪管理、职业目标
- 优势：能看到内心真实想法

### 其他文本
- 根据内容灵活提取
- 如果文本与工作无关，标注"无数据"

---

## 时间权重机制

与感性层相同，工作块分析也需要按时间分层：

| 时间段 | 权重 | 用途 |
|-------|------|------|
| 近 1 个月 | 100% | 当前工作状态 |
| 1-3 个月前 | 80% | 近期工作状态 |
| 3-6 个月前 | 50% | 中期参考 |
| 6-12 个月前 | 25% | 远期参考 |
| 1-3 年前 | 10% | 识别职业发展轨迹 |
| 3 年以上 | 5% | 识别底层职业价值观 |

**特别注意**：
- 职业信息变化快（跳槽、转岗、晋升），以最新为准
- 技能学习有时效性，旧技能可能已废弃
- 职业目标会随时间调整，以近期表达为准

**对于无时间戳的文本（书籍/文章）**：
- 假设文本代表作者当前状态
- 如果有多个时期的作品，可对比分析职业发展

## 分析维度

### 第一部分：职业现状

#### 1. 基础职业信息

**公司与行业**
- 当前公司：{从聊天中识别公司名称或暗示}
- 行业领域：{互联网/金融/教育/医疗/制造等}
- 公司规模：{大厂/中小厂/创业公司/外企}
- **输出格式**：`公司：[具体名称或"某互联网大厂"] - 行业：[具体领域]`

**职位与职级**
- 职位名称：{从聊天中识别}
- 职级：{如有明确提及}
- 团队规模：{是否管理团队，团队多大}
- **输出格式**：`职位：[具体职位] - 职级：[如有] - 管理：[是/否]`

**工作年限**
- 当前公司工作时长：{从聊天时间线推断}
- 总工作年限：{如有提及}
- **输出格式**：`当前公司：[X年Y月] - 总工作年限：[约X年]`

#### 2. 工作内容

**主要职责**
- 从聊天中识别 TA 日常在做什么
- 关键词：项目、需求、开发、设计、运营、管理等
- **输出格式**：`主要职责：[具体描述，3-5条]`

**技术栈/工具栈**
- 编程语言：{从聊天中识别提及的语言}
- 框架/库：{从聊天中识别}
- 工具：{IDE、协作工具、设计工具等}
- 基础设施：{数据库、云服务、中间件等}
- **输出格式**：
```
技术栈：
  - 编程语言：[Python, JavaScript, ...]
  - 框架：[React, Django, ...]
  - 工具：[VS Code, Figma, ...]
  - 基础设施：[MySQL, Redis, AWS, ...]
```

**项目类型**
- TA 参与的项目类型（ToB/ToC/内部系统/算法研究等）
- 项目规模（个人/小团队/大型项目）
- **输出格式**：`项目类型：[具体描述]`

#### 3. 工作习惯

**时间管理**
- 工作时段：{从聊天时间戳推断工作时间}
- 加班频率：{从深夜/周末聊天推断}
- 工作节奏：{从聊天内容推断是忙碌/轻松/波动}
- **输出格式**：`工作时段：[9:00-18:00 / 弹性 / 经常加班] - 节奏：[忙碌/轻松/波动]`

**协作模式**
- 独立工作 vs 团队协作：{从聊天中识别}
- 沟通频率：{是否经常提到开会、讨论、对接}
- 跨部门协作：{是否涉及多部门}
- **输出格式**：`协作模式：[独立为主/团队协作] - 沟通频率：[高/中/低]`

**任务处理方式**
- 计划性 vs 临时性：{是否提前规划还是临时应对}
- 并行任务能力：{是否同时处理多个任务}
- Deadline 处理：{从聊天中识别对 deadline 的态度}
- **输出格式**：`任务处理：[计划型/应急型] - 并行能力：[强/弱] - Deadline态度：[具体描述]`

**问题解决方式**
- 遇到技术问题的典型反应：{自己查/问同事/Google/ChatGPT}
- 学习新知识的方式：{看文档/看视频/实践/问人}
- 调试习惯：{从聊天中识别}
- **输出格式**：`问题解决：[具体方式] - 学习方式：[具体方式]`

#### 4. 工作满意度

**满意的方面**
- 从聊天中识别 TA 对工作的正面评价
- 关键词：喜欢、有意思、学到东西、团队好等
- **输出格式**：`满意点：[具体描述，引用原文]`

**不满的方面**
- 从聊天中识别 TA 的抱怨和不满
- 关键词：累、无聊、没意思、钱少、内卷、PUA等
- **输出格式**：`不满点：[具体描述，引用原文]`

**离职倾向**
- 是否有离职想法：{从聊天中识别}
- 离职原因：{如有明确提及}
- **输出格式**：`离职倾向：[强/中/弱/无] - 原因：[如有]`

#### 5. 技能水平评估

**技术能力**
- 从聊天中推断 TA 的技术水平
- 依据：解决问题的方式、讨论的深度、是否被请教
- **输出格式**：`技术能力：[初级/中级/高级/专家] - 依据：[具体描述]`

**软技能**
- 沟通能力：{从对话质量评估}
- 文档能力：{是否提到写文档，写得如何}
- 管理能力：{如有管理职责}
- 影响力：{是否能推动事情}
- **输出格式**：`软技能：沟通[强/中/弱] 文档[强/中/弱] 管理[强/中/弱/无]`

**领域知识**
- 业务理解深度：{从讨论业务的内容判断}
- 行业经验：{从聊天中识别}
- **输出格式**：`领域知识：[具体描述]`

### 第二部分：理想现状

#### 1. 职业目标

**短期目标（1年内）**
- 从聊天中识别 TA 近期想达成的目标
- 关键词：想学、打算、准备、计划等
- **输出格式**：`短期目标：[具体描述，引用原文]`

**中期目标（1-3年）**
- 从聊天中识别 TA 的中期规划
- 关键词：以后想、希望、目标是等
- **输出格式**：`中期目标：[具体描述，引用原文]`

**长期目标（3年以上）**
- 从聊天中识别 TA 的长期愿景
- 关键词：理想、梦想、最终等
- **输出格式**：`长期目标：[具体描述，引用原文]`

**如果没有明确目标**
- 标注为"未明确表达"
- 从聊天中推断可能的方向（但标注为推断）
- **输出格式**：`职业目标：未明确表达 - 可能方向：[推断，需标注]`

#### 2. 期待的转型方向

**职位转型**
- 是否想换职位：{从聊天中识别}
- 想换到什么职位：{如有提及}
- **输出格式**：`职位转型：[想换/不想换] - 目标：[如有]`

**行业转型**
- 是否想换行业：{从聊天中识别}
- 想换到什么行业：{如有提及}
- **输出格式**：`行业转型：[想换/不想换] - 目标：[如有]`

**技术方向转型**
- 是否想换技术方向：{从聊天中识别}
- 想学什么新技术：{如有提及}
- **输出格式**：`技术转型：[想换/不想换] - 目标：[如有]`

**管理 vs 技术路线**
- TA 倾向于走管理路线还是技术路线
- 从聊天中识别态度
- **输出格式**：`路线选择：[管理/技术/未定] - 态度：[具体描述]`

#### 3. 想学的技能

**正在学习**
- 从聊天中识别 TA 当前正在学的东西
- 关键词：在学、最近看、正在研究等
- **输出格式**：`正在学习：[具体技能/知识] - 进度：[如有提及]`

**计划学习**
- 从聊天中识别 TA 计划学但还没开始的
- 关键词：想学、打算学、准备学等
- **输出格式**：`计划学习：[具体技能/知识] - 原因：[如有提及]`

**感兴趣但未行动**
- 从聊天中识别 TA 表达兴趣但没有学习计划的
- 关键词：感觉不错、挺有意思、有机会试试等
- **输出格式**：`感兴趣：[具体技能/知识] - 兴趣程度：[高/中/低]`

#### 4. Gap 分析（现状 vs 理想）

**技能 Gap**
- 对比当前技能和目标技能，找出差距
- **输出格式**：
```
技能差距：
  - 当前：[具体技能列表]
  - 目标：[具体技能列表]
  - 缺失：[需要学习的技能]
```

**经验 Gap**
- 对比当前经验和目标经验，找出差距
- **输出格式**：
```
经验差距：
  - 当前：[具体经验]
  - 目标：[具体经验]
  - 缺失：[需要积累的经验]
```

**推荐学习路径**
- 基于 Gap 分析，推荐学习路径
- 优先级排序：紧急且重要 > 重要不紧急 > 紧急不重要
- **输出格式**：
```
推荐学习路径：
  1. [技能A] - 优先级：高 - 原因：[具体原因]
  2. [技能B] - 优先级：中 - 原因：[具体原因]
  3. [技能C] - 优先级：低 - 原因：[具体原因]
```

## 注意事项

1. **区分事实和推断**：
   - 事实：聊天中明确提到的信息
   - 推断：从聊天中间接推断的信息，必须标注"推断"

2. **时间敏感性**：
   - 职业信息变化快，以最新为准
   - 如果发现跳槽、转岗，要明确标注时间节点

3. **技能评估的依据**：
   - 不要凭空评估，必须有聊天记录支撑
   - 引用具体对话作为证据

4. **目标的真实性**：
   - 区分"随口说说"和"认真规划"
   - 如果只是闲聊提到，标注为"兴趣"而非"目标"

5. **Gap 分析的实用性**：
   - 推荐学习路径要具体可行
   - 考虑 TA 的时间、精力、当前水平

6. **工作满意度的客观性**：
   - 不要过度解读抱怨（可能只是发泄）
   - 综合多次对话判断真实态度

7. **隐私保护**：
   - 如果涉及公司机密、敏感项目，脱敏处理
   - 保留特征，隐藏具体信息

8. **数据不足的处理**：
   - 如果某个维度聊天记录中完全没有，标注"无数据"
   - 不要为了完整性而编造信息
