# 克隆.skill 创建器

## 触发条件

当用户说以下任意内容时启动：
- `/clone`
- "帮我创建一个克隆 skill"
- "我想克隆一个人"
- "生成 XX 的克隆体"

当用户对已有克隆 Skill 说以下内容时，进入进化模式：
- "我有新资料" / "追加数据"
- "这里不像 TA" / "TA 不会这样说"
- `/update-clone {slug}`

当用户说 `/list-clones` 时列出所有已生成的克隆体。

---

## 工具使用规则

本 Skill 运行在 Claude Code 环境，使用以下工具：

| 任务 | 使用工具 |
|------|---------|
| 读取任意文本文件 | `Read` 工具 |
| 解析聊天记录 | `Bash` → `python3 ${CLAUDE_SKILL_DIR}/tools/chat_parser.py` |
| 解析通用文本 | `Bash` → `python3 ${CLAUDE_SKILL_DIR}/tools/text_parser.py` |
| 分析感性层 | 参考 `${CLAUDE_SKILL_DIR}/prompts/emotion_extractor.md` |
| 分析工作块 | 参考 `${CLAUDE_SKILL_DIR}/prompts/work_analyzer.md` |
| 分析生活块 | 参考 `${CLAUDE_SKILL_DIR}/prompts/life_analyzer.md` |
| 写入 Skill 文件 | `Write` / `Edit` 工具 |
| 列出已有 Skill | `Bash` → `ls -la ${CLONE_DIR}` |

**基础目录**：Skill 文件写入 `./clones/{slug}/`（相对于本项目目录）。

## 主流程：创建新克隆 Skill

### Step 1：基础信息录入

只问 2 个问题：

1. **对象名字**（必填）
   - 示例：`玮炜` / `张三` / `鲁迅` / `乔布斯`

2. **文本资料文件路径**（必填）
   - 示例：`/Users/xxx/Downloads/私聊_玮炜.txt`
   - 支持格式：
     - 聊天记录（微信/QQ/Telegram 导出的 txt）
     - 书籍（txt/md/pdf）
     - 文章/博客（txt/md）
     - 日记（txt/md）
     - 社交媒体内容（txt）
     - 邮件（txt/eml）
     - 其他任意文本

确认信息无误后进入下一步。

---

### Step 2：解析文本资料

**自动识别文本类型**，使用对应解析器：

#### 2.1 如果是聊天记录

使用 `chat_parser.py`：

```bash
python3 ${CLAUDE_SKILL_DIR}/tools/chat_parser.py \
  --input {file_path} \
  --target-name {name} \
  --output-dir ./clones/{slug}/knowledge
```

展示统计信息：
- 消息总数
- 时间跨度
- 时间权重分布
- 平均消息长度
- 平均回复速度

#### 2.2 如果是其他文本（书籍/文章/日记等）

使用 `text_parser.py`：

```bash
python3 ${CLAUDE_SKILL_DIR}/tools/text_parser.py \
  --input {file_path} \
  --target-name {name} \
  --output-dir ./clones/{slug}/knowledge
```

展示统计信息：
- 文本类型（自动识别）
- 总字数
- 总段落数

询问用户是否继续。

### Step 3：特征分析（四层并行）

**重要**：以下四个分析可以并行进行，互不依赖。

**分析策略根据文本类型调整**：

#### 文本类型：聊天记录
- 感性层：重点分析语言习惯、表达节奏、情绪模式
- 工作块：从工作相关对话提取
- 生活块：从日常对话提取
- 理性层：从问题解决、决策过程提取

#### 文本类型：书籍/文章
- 感性层：分析写作风格、用词偏好、论述方式
- 工作块：从内容主题、专业领域提取
- 生活块：从个人经历、价值观表达提取
- 理性层：从思维方式、论证逻辑提取

#### 文本类型：日记
- 感性层：分析情绪表达、自我对话方式
- 工作块：从工作记录提取
- 生活块：从日常记录提取
- 理性层：从反思、总结提取

#### 文本类型：其他
- 根据内容灵活调整，优先提取可识别的特征
- 数据不足的维度标注"无数据"

## 进化模式：追加新数据

当用户说"我有新的聊天记录"时：

1. 询问新聊天记录的文件路径
2. 使用 `chat_parser.py` 解析新记录
3. 对比新旧记录的时间跨度，判断是否有重叠
4. 重新计算时间权重（新记录权重更高）
5. 重新运行四层分析
6. 使用 `Edit` 工具增量更新各个特征文件
7. 更新 `meta.json` 的 `updated_at` 和 `version`
8. 保存旧版本到 `./clones/{slug}/versions/v{n}/`

## 进化模式：对话纠正

当用户说"这里不像 TA" / "TA 不会这样说"时：

1. 询问具体哪里不像

3. 生成纠正记录
4. 使用 `Edit` 工具在对应文件末尾追加：

```markdown
---

## 用户纠正记录

### 纠正 1 - {timestamp}
**原特征**：{具体描述}
**纠正为**：{用户提供的正确描述}
**影响范围**：{具体哪些行为规则需要调整}
```

5. 重新生成 `SKILL.md`
6. 更新 `meta.json` 的 `corrections_count` 和 `updated_at`

## 注意事项

### 1. 时间权重的应用

所有分析都必须遵循时间权重机制：
- 近 1 个月：100% 权重
- 1-3 个月：80% 权重
- 3-6 个月：50% 权重
- 6-12 个月：25% 权重
- 1-3 年：10% 权重
- 3 年以上：5% 权重

**永久特征 vs 阶段性特征**：
- 永久特征：所有时间段一致 → 直接纳入核心特征
- 阶段性特征：有变化 → 以最新为准，旧特征标注为"历史状态"

### 2. 数据质量评估

分析前评估数据质量：
- 消息总数 < 100：数据不足，提示用户
- 时间跨度 < 30 天：时间太短，提示用户
- 某个时间段消息为 0：提示数据断层

### 3. 置信度标注

根据数据量和质量，标注每层特征的置信度：
- **高置信度**：数据充足（>500 条），时间跨度长（>180 天），特征明显
- **中置信度**：数据一般（100-500 条），时间跨度中等（30-180 天）
- **低置信度**：数据不足（<100 条），时间跨度短（<30 天），或特征不明显

低置信度的特征，在生成时应更保守，避免过度推断。

### 4. 隐私保护

- 生成的 Skill 中，脱敏处理敏感信息（公司名、具体项目名、他人姓名）
- 保留特征，隐藏具体细节
- 如果用户明确要求保留，则保留

### 5. 推断的边界

- 明确区分"事实"和"推断"
- 事实：聊天中明确提到的
- 推断：从聊天中间接推断的，必须标注"推断"或"可能"
- 不确定的信息，标注"从聊天记录中无法判断"

### 6. 矛盾处理

当新旧记录出现矛盾时：
- 不要取平均
- 标注特征发生了变化
- 以近期为准描述当前特征
- 保留历史特征作为背景信息

### 7. 文件组织

```
clones/
└── {slug}/
    ├── SKILL.md              # 最终可运行的 skill
    ├── meta.json             # 元数据
    ├── features/             # 特征层（中间产物）
    │   ├── life.md
    │   ├── work.md
    │   ├── emotion.md
    │   └── rational.md
    ├── knowledge/            # 原始数据
    │   ├── {name}_parsed.json
    │   └── {name}_chat.md
    └── versions/             # 历史版本
        ├── v1/
        ├── v2/
        └── ...
```

```
当前支持的格式：
  - 微信 txt 导出（格式：时间戳 + 发送者 + 内容）

示例：
2025-04-11 10:42:15 '玮炜'
哈哈哈

如果你的聊天记录是其他格式，请提供样例，我可以适配。
```

### 数据量不足

如果消息数 < 100，提示用户：
```
⚠️ 数据量不足

当前消息数：{count} 条
建议消息数：>500 条

数据量不足可能导致：
  - 特征提取不准确
  - 置信度低
  - 克隆效果不理想

是否继续？
```

### 目标对象消息为 0

如果筛选后目标对象的消息为 0，提示用户：
```
❌ 未找到目标对象的消息

可能原因：
  1. 对象名字输入错误（请检查聊天记录中的名字）
  2. 聊天记录中没有该对象的消息

请重新输入对象名字。
```
